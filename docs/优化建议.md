# v10.3 代码与体验优化说明

这版主要不是继续堆功能，而是整理结构，让后面继续开发时不容易乱。

## 已经做的优化

### 1. 模块编辑器事件简化
旧版每次渲染模块都会给每个按钮、输入框重新绑定一堆事件，模块多了之后会越来越卡。  
这版改成了事件委托：每个模块列表只绑定一次点击 / 输入事件，后续新增、删除、复制模块都复用这套逻辑。

### 2. 实时预览防抖
文章和页面右侧预览从“每输入一个字立即渲染”改为轻量防抖渲染。  
输入时仍然实时，但不会每个键都立刻重绘，模块多、图片多时更流畅。

### 3. 模块折叠
模块多了之后页面会很长，现在每个模块都可以折叠。  
适合长文章：只展开正在编辑的模块，其余折叠起来。

### 4. 模块复制
增加“复制模块”，常见场景比如多个商品卡、多个 FAQ、多个步骤模块，不用重复手动建。

### 5. 未保存离开提醒
文章、页面、站点设置修改后，如果没有保存就刷新 / 关闭页面，会弹出提醒，避免内容丢失。

### 6. 插入动画轻量化
模块插入动画保留，但只做轻量 transform / opacity，不做全屏遮罩和重绘型动画。

## 后续建议

### A. 把模块编辑器拆成独立文件
现在 `public/js/admin.js` 仍然比较大，建议后续拆成：

```text
public/js/admin-core.js
public/js/admin-modules.js
public/js/admin-preview.js
public/js/admin-settings.js
public/js/admin-taxonomy.js
```

这样以后维护更清楚。

### B. 模块数据建议从 Markdown 升级为 JSON
现在模块保存时会转成 Markdown，兼容性好，但复杂模块会越来越难维护。  
后续可以新增字段：

```text
content_blocks JSON
```

这样每个模块完整保存，预览和前台渲染都更稳定。

### C. 自定义 CSS 建议加白名单
现在给了全站自定义 CSS，灵活但也有风险。  
个人站可以用，后期多人后台建议限制 CSS 注入权限。

### D. 图片建议自动压缩
现在支持本地上传，但还没做自动压缩。  
后续可以在前端上传前压缩成 WebP，免费存储会更耐用。

### E. 视频模块建议加平台模板
可以做成：

```text
B站 BV号
抖音分享链接
西瓜视频链接
YouTube 链接
```

用户填链接，系统自动生成 iframe，降低出错率。
