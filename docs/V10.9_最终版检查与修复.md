# V10.9 最终版检查与修复

这版是收尾正式版，重点不是继续堆功能，而是修复容易出问题的地方。

## 已修复

1. **手机端模块开关同步**
   - 原先移动端分类、标签、快捷导航、作者卡片在部分情况下不会跟随后台“模块显示开关”隐藏。
   - 已拆分移动端分类卡片和标签卡片，并让前台 JS 同时兼容桌面端和手机端容器。

2. **后台站点设置预览的初始化风险**
   - 站点设置右侧主页预览在初始化时可能因为函数声明顺序触发异常。
   - 已改成统一调度函数，避免首次登录后台时预览影响整体 JS。

3. **文章 / 页面 clean URL 冲突**
   - 文章和页面如果设置成同一个 slug，会导致页面优先覆盖文章。
   - 已在后端做跨表去重：文章 slug 会避开页面 slug，页面 slug 会避开文章 slug。

4. **保留关键路由**
   - `api`、`archive`、`category`、`tag`、`search`、`uploads` 等保留路径不会被文章或页面 slug 直接占用。

5. **API 缓存问题**
   - `/api/*` 统一加了 no-store 头，避免站点设置保存后前台仍读到旧接口结果。

6. **评论安全增强**
   - 评论验证码错误现在返回 400，不再被当成服务器 500 错误。
   - 新增简单 IP 频率限制：10 分钟内最多 8 条评论，减少刷屏风险。

7. **系统检查模块**
   - 后台新增“系统检查”页，可以查看数据库模式、图片存储配置是否存在、上传大小、后台入口等。
   - 不会显示 service role key 明文。

8. **Render 部署配置优化**
   - `render.yaml` 已改为 pnpm 构建命令。
   - 固定 Node 20.18.1，并加入 `.nvmrc`。
   - 删除 `package-lock.json`，避免再次误用 `npm ci` 导致安装异常。

## Render 推荐配置

Build Command：

```bash
npm install -g pnpm@9.12.3 && pnpm install --no-frozen-lockfile
```

Start Command：

```bash
node src/server.js
```

Node：

```text
20.18.1
```

## 更新提醒

如果你用 GitHub 网页覆盖新版，请记得删除旧的 `package-lock.json`，并保留：

```text
.node-version
.nvmrc
package.json
src
public
private
```
